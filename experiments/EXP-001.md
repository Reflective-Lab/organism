# EXP-001: Formation tournament + prior learning uses only existing types

**Date:** 2026-04-21
**Status:** confirmed
**Hypothesis:** Organism can run multiple competing Formations for the same intent, compare their `ConvergeResult`s using Converge's `ScoredFormation` scoring, and feed the winner/loser signal back through the learning adapter so `PlanningPriorAgent` calibrates future formation selection — all using existing types across both crates without new Converge dependencies or wrapper layers.
**Falsification criterion:** If running two Formations with different agent teams on the same seed, scoring both results, and feeding the winner/loser delta into `calibrate_priors()` requires new Converge types or violates the no-wrapper rule — the hypothesis fails.
**Method:** Implement `FormationTournament` in `organism-runtime` using only `Formation`, `FormationResult`, `adapter::build_episode_from_run`, and `adapter::calibrate_priors`. Score results by `ConvergeResult` fields (`converged`, `cycles`, `criteria_outcomes`). Verify all tests pass with `just test`. Confirm no new Converge crate dependencies are added.

## Setup

Add `FormationTournament` to `crates/runtime/src/formation.rs` (or a new `tournament.rs`). It must:
1. Accept N `Formation`s and run them concurrently via `tokio::join` or sequential `run()` calls.
2. Score each `FormationResult` from `ConvergeResult` fields alone.
3. Return a `TournamentResult` with winner label, scores, and a `PriorCalibration` vec ready for the next `PlanningPriorAgent` seed.

No new Converge crate deps. No wrapper traits.

cargo test -p organism-runtime --lib

## Evidence

- `crates/runtime/src/tournament.rs` added — 386 lines, no new Converge crate dependencies.
- `Cargo.toml` delta: only `uuid.workspace = true` added (already in workspace, needed for `Uuid` IDs).
- 8 new tests in `tournament::tests`, all pass alongside existing 30 runtime tests (38 total).
- `FormationTournament::run()` uses only: `Formation::run()` → `FormationResult` → `ConvergeResult` fields (`converged`, `cycles`, `criteria_outcomes`) for scoring. Zero new Converge types.
- `calibrate_priors()` from `organism_learning::adapter` accepted a `LearningEpisode` built from tournament scores without modification.
- Priors serialise as valid `PlanningPriorAgent` seeds (verified by `priors_are_serializable_as_planning_prior_seeds` test).
- `repeated_tournaments_tighten_priors` confirmed evidence count increments across rounds: round 1 = 1, round 2 = 2.
- No wrapper traits over Converge. `Formation` and `FormationResult` used directly.

## Outcome

**Confirmed.**

The hypothesis holds: the tournament + learning loop is expressible entirely with existing organism and Converge types. `FormationTournament` runs N competing formations, scores them from `ConvergeResult` fields, and feeds the winner's score profile into `calibrate_priors()` as three prediction-error dimensions (`convergence_rate`, `criteria_coverage`, `cycle_efficiency`). Those priors are valid `PlanningPriorAgent` seeds, closing the loop. No new Converge crate dependencies, no wrapper layers. The one open gap from the original idea — the "formation guru" that selects which agents to put in a formation based on priors — is not yet implemented; `FormationTournament` assumes the caller assembles the formations. That selection logic is the natural next step.
