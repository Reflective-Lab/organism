# Organism Examples

The examples here are not all trying to prove the same thing.

Some demonstrate the straight pipeline. Others demonstrate the odd formation
patterns that only make sense once you accept that the team shape itself can be
derived, adapted, or discovered.

## Core Loops

| Example | What it proves |
|---|---|
| [`vendor-selection`](vendor-selection) | Multi-criteria swarm evaluation with domain metadata |
| [`expense-approval`](expense-approval) | Full pipeline from admission through simulation |
| [`loan-application`](loan-application) | Parallel evaluation, skepticism, simulation, and learning capture |
| [`debate-loop`](debate-loop) | Planner and skeptic revising each other inside one converge loop |
| [`due-diligence-loop`](due-diligence-loop) | Self-organizing `Formation` that seeds diligence strategies, reopens gaps, and only synthesizes at stability |

## Formation and Shape

| Example | Why it matters |
|---|---|
| [`collab-huddle`](collab-huddle) | The strict default when explicit turns, dissent tracking, and done gates matter |
| [`collab-panel`](collab-panel) | Curated expert formation with explicit role enforcement |
| [`collab-discussion`](collab-discussion) | Moderated, lighter-weight discussion topology |
| [`collab-self-organizing`](collab-self-organizing) | Start loose when you do not yet know the right cast |
| [`charter-from-intent`](charter-from-intent) | Derive collaboration shape from intent complexity instead of hand-picking it |
| [`topology-transition`](topology-transition) | Tighten or reshape collaboration as contradictions, stability, or budget pressure change |
| [`shape-competition`](shape-competition) | Treat the collaboration shape itself as a hypothesis and learn which topology works |

## Non-Obvious Formations

These are the examples to read when the normal “huddle first, debate later”
story is too tidy.

- [`collab-self-organizing`](collab-self-organizing): useful when the search
  space is unclear and over-curating the team too early would make the loop
  dumber.
- [`topology-transition`](topology-transition): useful when the right shape
  changes mid-run, especially under contradiction spikes or budget pressure.
- [`shape-competition`](shape-competition): useful when you do not actually know
  the best collaboration mode ahead of time and want evidence, not taste.
- [`due-diligence-loop`](due-diligence-loop): useful when a loose open-call team
  should seed the first passes, but the work itself keeps discovering missing
  information and the loop must reopen until coverage is good enough.

Those are not novelty demos. They are the reason Organism exists above Converge.
